# WanderAI - Frontend Development Summary

This document outlines the entire frontend development process for the WanderAI project, detailing the architecture, libraries used, and major features implemented.

## 1. Technologies & Libraries
The frontend was built as a Single Page Application (SPA) utilizing modern web technologies:
- **React 19** (via Create React App): The core framework for building the user interface.
- **react-router-dom**: Used for handling client-side routing between the different pages (`/`, `/chat`, `/saved`) without reloading the browser.
- **lucide-react**: A professional, clean SVG icon library. This replaced earlier emoji usage to give the application a much more polished and sophisticated feel.
- **Vanilla CSS**: Used extensively with advanced features like Custom Properties (Variables), Flexbox/Grid layouts, CSS animations, and `backdrop-filter` for glassmorphic effects.

## 2. Core Architecture & Pages
The application is structured into three primary pages, all wrapped inside a `ThemeProvider` and `Router` in `App.js`.

- **Landing Page (`/`)**: A hero-driven page allowing users to type a natural language query and select a travel style. It features animated background gradients and showcases the app's capabilities.
- **Chat Page (`/chat`)**: A split-panel interface. The left side handles the conversational interface with typing indicators and auto-scrolling. The right side dynamically renders the AI-generated itinerary, budget breakdown, and seasonal alerts.
- **Saved Trips Page (`/saved`)**: A dashboard grid displaying previously generated itineraries fetched from the backend.

## 3. Reusable Components
To keep the codebase modular, several reusable components were created:
- **Navbar**: Features smooth glassmorphism, responsive mobile hamburger menu, and a dark mode toggle.
- **DayCard**: Displays individual day plans with structured time slots (Morning, Afternoon, Evening) and travel/stay metadata.
- **BudgetBreakdown**: A visual progress bar and categorized list showing exactly how the trip funds are allocated.
- **TripCard**: Used on the Saved Trips page to display summary information and images (via Unsplash) for saved itineraries.
- **Toast**: A custom, auto-dismissing notification system that slides in from the top right to provide user feedback (success, error, warning).
- **ConfirmModal**: A custom frosted-glass confirmation dialog that replaced the ugly default browser `window.confirm()` popup for deleting trips.
- **Skeleton**: Shimmer-effect loading placeholders displayed while waiting for the AI backend or database to respond.

## 4. Key Technical Fixes
- **Duplicate Request Bug Fixed**: Addressed an issue where React's Strict Mode caused the initial chat query to be sent twice. Implemented a `useRef` based guard (`querySentRef`) to ensure queries only trigger once.
- **Routing State Optimization**: Fixed a bug where clicking a saved trip would generate and save a duplicate trip. Changed the routing logic to pass the entire `tripData` object in the location state. The `ChatPage` was updated to read this local data and render the UI instantly instead of hitting the backend API again.

## 5. UI/UX Overhaul & Theming
The UI went through a major overhaul to transform it from a basic prototype to a premium product:
- **Design System (`index.css`)**: Established a robust system of CSS variables (tokens) for colors, spacing, typography, and shadows.
- **Dark Mode (`ThemeContext.js`)**: Implemented a global context provider to handle theming. It detects system preferences, allows manual toggling via the Navbar, and persists the choice in `localStorage`. All CSS components adapt seamlessly via `[data-theme="dark"]` selectors.
- **Glassmorphism & Spring Animations**: Implemented heavy use of `backdrop-filter: blur()` on overlays, modals, and navbars. Upgraded standard CSS transitions to use cubic-bezier curves (e.g., `cubic-bezier(0.16, 1, 0.3, 1)`) to give elements a snappy, spring-like feel when hovered or rendered.

## 6. Project Cleanup
- Removed unnecessary default React boilerplate files that were cluttering the workspace (`App.test.js`, `setupTests.js`, `reportWebVitals.js`, default React logos).
